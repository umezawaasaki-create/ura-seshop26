# URA SSP 2026 トーナメント対戦管理アプリ

「世小P裏 URA SSP 2026」トーナメント表をベースにした、対戦日時・グラウンドの登録と試合結果（得点）入力ができるWebアプリです。Google Apps Script + Google スプレッドシートで構築されており、追加のサーバーやホスティング費用は不要です。

## できること

- トーナメント表（ラウンド16〜準々決勝・準決勝・決勝・3位決定戦）をWeb上に表示
- 各試合カードをクリックして「対戦日時・グラウンド」を入力・保存
- 各試合カードから「得点」を入力し、勝者を自動判定して次のラウンドの対戦カードに反映（準決勝の敗者は3位決定戦に自動反映）
- 不戦勝（スコアなしでの勝ち上がり）にも対応
- すべてのデータはGoogleスプレッドシートの `Teams` / `Matches` シートに保存され、スプレッドシート側からも直接確認・修正が可能

アクセス制限は設けていません。WebアプリのURLを知っている人なら誰でも入力できる簡易運用を想定しています。

## ファイル構成

```
gas/
├── appsscript.json    マニフェスト（タイムゾーン、Webアプリ公開設定）
├── Code.gs             サーバーサイドロジック（データ読み書き・勝者自動進出）
├── Index.html          ページ本体
├── CSS.html             スタイル
├── JS.html              クライアントスクリプト（描画・モーダル操作）
├── logo.png             中央ロゴの元画像（JS.html内に埋め込み済み。差し替え時の参照用）
├── title.png             ヘッダータイトルの画像（Index.html内に埋め込み済み。差し替え時の参照用）
├── preview_mock.html   ブラウザで直接開ける見た目確認用モック（下記参照）
└── README.md            このファイル
```

### 見た目だけ先に確認したいとき

`preview_mock.html` をブラウザでダブルクリックして開くと、Googleスプレッドシートに接続せず、ダミーデータでブラケットの見た目と操作感（試合クリック→モーダル→保存→勝者の自動進出）だけをその場で確認できます。実際のデプロイ前の確認用で、保存内容は保存されません（ページを再読み込みすると初期状態に戻ります）。実際の運用にはデプロイ手順に沿ってGoogle Apps Script側を必ずセットアップしてください。

## セットアップ手順

### 1. スプレッドシートを新規作成し、Apps Scriptを開く

1. Google スプレッドシートを新規作成（例: 「URA SSP 2026 トーナメント管理」）
2. メニューの **拡張機能 → Apps Script** を開く（このシートに紐づくスクリプトプロジェクトが開きます）

### 2. コードを配置する

**方法A: clasp を使う場合（推奨・自分のPCのターミナルで実行）**

```bash
npm install -g @google/clasp
clasp login                     # ブラウザでGoogleアカウント認証
```

開いたApps ScriptエディタのURL末尾にあるスクリプトID（`https://script.google.com/.../projects/<SCRIPT_ID>/edit`）を控え、このリポジトリの `gas/` フォルダに移動して:

```bash
cd gas
clasp clone <SCRIPT_ID> --rootDir .
```

`.clasp.json` が生成されるので、その後は以下でこのフォルダの内容をアップロードできます。

```bash
clasp push
```

**方法B: コピー&ペーストする場合**

1. Apps Scriptエディタで `Code.gs` の中身をこのリポジトリの [Code.gs](Code.gs) の内容で置き換える
2. エディタ左側の「+」から HTML ファイルを3つ追加し、それぞれ `Index`・`CSS`・`JS` という名前にして、対応する内容を貼り付ける
   - [Index.html](Index.html)
   - [CSS.html](CSS.html)
   - [JS.html](JS.html)
3. プロジェクトの「プロジェクトの設定」→「`appsscript.json` マニフェストファイルをエディタで表示する」を有効にし、[appsscript.json](appsscript.json) の内容で置き換える

### 3. 初期データを投入する

1. Apps Scriptエディタ上部の関数選択で `setupSheet` を選び、実行（▶）ボタンを押す
2. 初回は権限確認画面が出るので許可する
3. スプレッドシートに `Teams`・`Matches` の2シートが作成され、画像のトーナメント表の内容（16チーム、既に確定している「玉川 不戦勝」「8/23（若林 vs 深沢）」「7/18（駒繋 vs 多聞）」を含む）が入っていることを確認する

※ `setupSheet` は実行するたびに `Teams`/`Matches` シートの内容を初期状態にリセットします。試合が進行した後は誤って再実行しないよう注意してください。

### 4. Webアプリとしてデプロイする

1. Apps Scriptエディタ右上の「デプロイ」→「新しいデプロイ」
2. 種類の選択で歯車アイコン→「ウェブアプリ」を選択
3. 「次のユーザーとして実行」: **自分**
4. 「アクセスできるユーザー」: **全員**
5. 「デプロイ」をクリックし、発行されたWebアプリのURLを控える
6. そのURLにアクセスし、トーナメント表が表示されることを確認する

コードを更新した場合は、`clasp push` の後に「デプロイ」→「デプロイを管理」→ 既存のデプロイの編集（鉛筆アイコン）→ バージョン「新バージョン」を選んで更新してください（URLは変わりません）。

## データの見方（Matches シート）

| 列 | 内容 |
|---|---|
| match_id | 試合ID（例: `R16-1`, `QF-1`, `SF-1`, `F`, `3RD`） |
| team1_id / team2_id | 対戦チーム（`Teams`シートの team_id） |
| date / time / venue | 対戦日・時刻・グラウンド |
| score1 / score2 | 得点 |
| winner_id | 勝者（自動設定） |
| win_type | `normal`（通常）/ `fusen1`・`fusen2`（不戦勝） |
| next_match_id / next_slot | 勝者が進む次の試合IDと枠（1 or 2） |
| loser_next_match_id / loser_next_slot | （準決勝のみ）敗者が進む3位決定戦の試合IDと枠 |

試合結果や日程はWebアプリからだけでなく、スプレッドシートを直接編集しても反映されます（ただし勝者の自動進出はWebアプリの「結果を保存」経由でのみ実行されます。スプレッドシートを直接書き換えた場合は次戦の対戦チームを手動で入力してください）。

## チームアイコンをアプリから直接アップロードする

画面左上のハンバーガーメニュー（&#9776;）を開くと、全チームがあいうえお順（`Teams` シートの `kana` 列の読みに基づく）で一覧表示されます。チーム名は一覧上では変更できません。アイコン（バッジ）をタップすると画像変更モーダルが開き、「画像を選択」から画像ファイルを選ぶと、自動的にGoogleドライブ（`URA_SSP_2026_チームアイコン` フォルダ、共有設定「リンクを知っている全員が閲覧可」で作成されます）にアップロードされ、`Teams` シートの `logo_url` に反映されます。「元に戻す」で色付きイニシャルのバッジに戻せます。

画像サイズは5MBまでです。初回利用時、Googleドライブへのアクセス許可の再承認が求められることがあります。

## チームロゴを差し替える（Teams シート、手動）

上記のアップロード機能を使わず、`Teams` シートの `logo_url` 列に直接画像のURLを入力することもできます（空欄のままなら、これまで通り色付きイニシャルのバッジが表示されます）。

| 列 | 内容 |
|---|---|
| team_id | チームID（Matchesシートから参照される） |
| name | チーム名 |
| initial | logo_urlが空のときに表示するイニシャル |
| color | logo_urlが空のときのバッジの背景色 |
| logo_url | ロゴ画像のURL（空欄でも可） |
| kana | チーム名の読み（ひらがな）。ハンバーガーメニューのチーム一覧をあいうえお順に並べるためだけに使用（空欄なら name をそのまま比較） |

画像を用意する簡単な方法:
1. Googleドライブに画像をアップロードし、共有設定を「リンクを知っている全員」にする
2. ファイルを右クリック→「リンクを取得」でURLをコピー（`https://drive.google.com/file/d/<ID>/view...`の形式）
3. そのままだと画像として直接表示できないため、`https://drive.google.com/uc?export=view&id=<ID>` の形式に組み替えて `logo_url` に貼り付ける（`<ID>` の部分は元のURL中のファイルIDに置き換える）

外部の画像ホスティングサービスの直リンクURL（`https://.../logo.png` など）でも問題なく表示されます。

## 大会ロゴ（トーナメント表中央のバッジ）を差し替える

決勝と3位決定戦の間に表示される大会ロゴは、外部URLではなく [JS.html](JS.html) の `LOGO_DATA_URI`（`trophyHtml()` の直前）に画像データを直接埋め込む形にしてあります。外部リンク切れの心配がなく、常に表示される安心感を優先したためです。

差し替えたい場合は、新しい画像を base64 に変換し、`LOGO_DATA_URI` の値（`data:image/png;base64,...`）をまるごと置き換えてください。ターミナルで以下のように変換できます。

```bash
python3 -c "
import base64
with open('新しい画像.png', 'rb') as f:
    print('data:image/png;base64,' + base64.b64encode(f.read()).decode())
"
```

参考として、現在使用している元画像を [logo.png](logo.png) として同梱しています。

## ヘッダーのタイトル画像を差し替える

ヘッダー上部の「KNOCKOUT STAGE / ROUND 16」の文字は、Webフォント（Anton）をライブ表示するのではなく、あらかじめ画像として書き出したものを [Index.html](Index.html) の `<h1>` 内に `<img>` として直接埋め込んでいます。実行環境によってフォントの見え方が変わらないよう、画像として固定しています。

差し替えたい場合は、新しい画像を上記と同じ方法でbase64に変換し、`Index.html` 内の `<img class="title-img" src="data:image/png;base64,...">` の値をまるごと置き換えてください。参考として、現在使用している元画像を [title.png](title.png) として同梱しています。
